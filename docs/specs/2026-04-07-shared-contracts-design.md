# Shared Contracts Design — YAGA-26

**Date:** 2026-04-07
**Work item:** YAGA-26
**Status:** Approved
**Parent epic:** YAGA-23 (Kickoff Foundation: Bootstrap and Contracts)

## Goal

Implement the contracts layer (`yaga_contracts` package) so that persistence
and services cannot invent incompatible schemas. All Pydantic DTOs for HTTP
request/response, event envelopes, handoff payloads, webhook callbacks, and
internal commands live here.

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Module layout | Flat files under `yaga_contracts/` | Small scope (~15 models), 300-line file limit, no need for subdirectories yet |
| Inheritance strategy | Hybrid — bases for commands and events only | Commands share a genuine envelope (7 fields); events share a genuine envelope (11 fields). HTTP and webhook DTOs are flat. |
| ID types | Mixed: `UUID` for internal, `str` for external | Internal IDs are generated by the runtime (strong validation). External IDs (webhook event_id, idempotency_key) come from outside (accept any format). |
| Event payloads | `dict[str, Any]` on envelope, typed payload models for deserialization | Avoids a 30+ type discriminated union that couples unrelated event domains. Consumers pick the payload model by `event_type`. |
| Pydantic config | `model_config = ConfigDict(frozen=True)` on all models | Contracts are immutable data containers. |

## File Layout

```
packages/contracts/yaga_contracts/
├── __init__.py          # Re-exports all public models
├── shared.py            # Enums, value objects (Actor, ReplyTarget)
├── http.py              # HTTP request/response DTOs
├── events.py            # Event envelope base + typed payload models
├── handoffs.py          # Handoff dispatch, response, completion DTOs
├── webhooks.py          # Webhook callback DTO + header constants
└── commands.py          # Command base + 10 concrete command types
```

## Module Specifications

**Enums:**

| Enum | Values |
|------|--------|
| `TaskStatus` | `CREATED`, `ACCEPTED`, `IN_PROGRESS`, `REVIEW`, `VERIFY`, `DONE`, `BLOCKED`, `ESCALATED`, `CANCELLED` |
| `HandoffStatus` | `DISPATCHED`, `ACCEPTED`, `REJECTED`, `TIMED_OUT` |
| `RequestStatus` | `RECEIVED`, `NORMALIZED`, `DELEGATED`, `AWAITING_CALLBACK`, `REPLY_PENDING`, `REPLY_PUBLISHED`, `REPLY_FAILED`, `FALLBACK_REQUIRED`, `CLOSED` |
| `PublishStatus` | `PENDING`, `ATTEMPTED`, `PUBLISHED`, `FAILED` |

| `ExecutionMode` | `SESSION_BOUND`, `DETACHED` |
| `HandoffResolution` | `ACCEPTED`, `REJECTED` |

| `JobStatus` | `QUEUED`, `RUNNING`, `COMPLETED`, `FAILED`, `CANCELLED` |

| Model | Fields |
|-------|--------|
| `Actor` | `actor_type: str`, `actor_id: UUID` |

### http.py — HTTP DTOs

| Model | Purpose | Fields |
|-------|---------|--------|
| `CreateRequestBody` | POST /requests body | `message: str`, `origin_channel: str`, `origin_session_key: str`, `reply_target: ReplyTarget \| None` |
| `CreateRequestResponse` | 202 response | `request_id: UUID`, `task_ref: UUID \| None`, `status: RequestStatus` |
| `RequestReadModel` | GET /requests/{id} | `request_id: UUID`, `correlation_id: UUID`, `status: RequestStatus`, `origin_channel: str`, `origin_session_key: str`, `strategic_owner_agent_id: UUID`, `current_owner_agent_id: UUID`, `reply_publish_status: PublishStatus`, `reply_target: ReplyTarget \| None`, `created_at: datetime`, `updated_at: datetime` |
| `ErrorResponse` | Error envelope | `error_code: str`, `message: str`, `request_id: UUID \| None` |

`Idempotency-Key` is a request header handled by the API layer, not part of the body DTO.

### events.py — Event Envelope and Payloads

**Base class — `EventEnvelope`:**

| Field | Type | Notes |
|-------|------|-------|
| `event_id` | `UUID` | Unique per emission, changes on retry |
| `dedup_key` | `UUID` | Stable across retries |
| `event_type` | `str` | e.g., `"request.received"` |
| `aggregate_type` | `str` | e.g., `"request"`, `"task"` |
| `aggregate_id` | `UUID` | |
| `correlation_id` | `UUID` | Required on all events |
| `causation_id` | `UUID \| None` | `None` on root events |
| `actor` | `Actor` | |
| `occurred_at` | `datetime` | |
| `schema_version` | `int` | Default `1` |
| `stream_sequence` | `int \| None` | Assigned by event store, not producer |
| `payload` | `dict[str, Any]` | Typed payload deserialized by consumers |

**Typed payload models:**

| Model | Key fields |
|-------|------------|
| `RequestReceivedPayload` | `message`, `origin_channel`, `origin_session_key`, `idempotency_key: str`, `request_class: RequestClass` |
| `RequestNormalizationPayload` | `request_id`, `strategic_owner_agent_id`, `default_assignee_agent_id` |
| `TaskLifecyclePayload` | `task_id`, `request_id`, `owner_agent_id`, `status: TaskStatus`, `outcome: TaskOutcome \| None` |
| `HandoffLifecyclePayload` | `handoff_id`, `task_id`, `from_agent_id`, `to_agent_id`, `status: HandoffStatus`, `reason: str \| None` |
| `CallbackPayload` | `task_id`, `source_agent_id`, `target_agent_id`, `result_summary: str \| None` |
| `PublicationPayload` | `publication_id`, `request_id`, `publish_dedup_key: str`, `status: PublishStatus`, `channel: str \| None`, `session_key: str \| None` |
| `WatchdogPayload` | `job_id`, `job_type: str`, `subject_type: str`, `subject_id` |
| `CommandRejectedPayload` | `command_type: str`, `reason: str` |

All UUID-typed fields in payloads use `UUID`. The `idempotency_key` and
`publish_dedup_key` are `str` (externally provided).

### __init__.py — Re-exports

All public models, enums, value objects, and constants are re-exported from
`yaga_contracts.__init__` so consumers import directly:

```python
from yaga_contracts import EventEnvelope, CreateRequestBody, HandoffDispatch
```

## Testing Strategy

Contract tests (Block H) will verify:
- All models serialize to JSON and deserialize back without data loss.
- Required fields reject `None` / missing values.
- UUID fields reject malformed strings.
- Enum fields reject invalid values.
- Frozen models reject attribute mutation.
- Event envelope `payload` round-trips through `dict`.
- Webhook header constants match expected string values.

## Acceptance Criteria

1. All DTOs serialize and validate per architecture docs.
2. Handoff and webhook models preserve required identifier distinctions
   (`event_id` vs `dedup_key`, `request_id` vs `correlation_id`,
   `publish_dedup_key` semantics).
3. Runtime-core services can consume internal DTOs rather than raw
   HTTP/persistence payloads.
4. Import-linter contracts pass (contracts remains a leaf package with no
   internal dependencies).
5. `scripts/lint.sh` passes with zero warnings.
6. `scripts/test.sh` passes with contract validation tests.
